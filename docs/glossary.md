# BookOS Glossary

This glossary mirrors the in-app help topics. The wording is intentionally short and action-oriented so new users can learn BookOS without reading implementation docs.

## Source Reference

A trace back to the book, note, capture, page, or excerpt that produced an idea. Source references keep BookOS honest. If a page is unknown, BookOS must say page unknown instead of inventing one.

## Quick Capture

The fastest way to save a raw reading thought while preserving book context. Example syntax: `💡 p.12 This could become a core loop [[Core Loop]] #prototype`.

## Capture Inbox

A processing queue for raw captures that still need conversion or concept review. Convert captures into notes, quotes, action items, or reviewed concepts.

## Quote

A source-backed passage or excerpt saved for later review. Quotes should reopen their source and can support daily resurfacing, search, and projects.

## Action Item

A practical task extracted from reading or project work. Action items turn reading into behavior, experiments, or design follow-through.

## Concept

A reviewed idea such as Core Loop, Game Feel, or Player Motivation. A `[[Concept]]` marker should be reviewed before it becomes durable knowledge.

## Knowledge Object

A structured knowledge record such as a principle, design lens, exercise, method, or pattern.

## Design Lens

A focused question or perspective for evaluating a game design problem.

## Daily Prompt

A daily question selected from source-backed knowledge or a clearly labeled template. Template prompts must not imply a book page.

## Project Application

A source-backed note explaining how reading knowledge applies to a game project.

## Design Decision

A recorded choice, rationale, and tradeoff for a project.

## Playtest Finding

An observation from project testing with severity, recommendation, and optional source link.

## Graph

A real-data map of relationships between books, notes, quotes, concepts, projects, and sources. The graph must not use decorative fake nodes.

## Backlink

A reverse relationship showing what else points to the current record.

## Review Session

A focused session for revisiting real source-backed items.

## Mastery

A user-owned familiarity and usefulness signal for a reviewed item. Mastery should reflect real review behavior, not fabricated analytics.

## AI Draft

An optional draft suggestion generated from selected BookOS content. AI drafts must never overwrite user content automatically.

## Import Preview

A non-writing safety check before imported records are saved. Preview is where duplicate, page, and source-reference issues should be surfaced.
